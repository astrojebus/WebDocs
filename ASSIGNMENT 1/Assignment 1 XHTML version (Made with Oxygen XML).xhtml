Description of Assignment

There were 3 main parts that constitute the graph: The grid-style to present a sense of scale, the x and y axis, and the triangle shape.

For the grid-style, a nested pattern was defined. The pattern element defines a graphics object which can be redrawn at repeated x and y-coordinate intervals ("tiled") to cover an area. Said pattern is referenced (using an id name) by the fill and/or stroke attributes on other graphics elements to fill or stroke those elements with the referenced pattern. In the assignment, one pattern of 10x10 gray colored cells are embedded in a main pattern of black colored cells, these will be replicated across the canvas space of the SVG. The patternUnits attribute indicates which coordinate system to use for the geometry properties of the pattern element. The userSpaceOnUse value indicates that all coordinates refer to the user coordinate system as defined when the pattern was applied, this means if the object changes size our pattern would stay the same, and just repeat itself more times to fill the box. The defs element is used to store graphical objects that will be used at a later time. Objects created inside a def element are not rendered directly. To display them one has to reference them (with their ID). The main pattern is referenced with the use of the fill attribute inside a rect element to fill the entire drawing canvas.

For the triangle and cell shapes the <path> is a powerful element that was used to draw the figure with the help of the the d attribute (The M indicates a moveto, the Ls indicate linetos, and the z indicates a closepath).

For the coordinates plane, the grouping "g" element was used to group graphical elements together, a feature not found in the SVG element, all the attributes included in this element are applied to child elements. Two subgroups are nested, one to draw the axis and a second one to draw it's numeral ticks. Once grouped all the shapes, the transform attribute is used to perform some presentation arrangement. The "scale" attribute changes the size of an item i.e 0.4 equals 40% of normal size , as we shrink the initial size of 1000x1000 by the translate attribute. The translate attribute will move the group in the x and y axis. Inside of the group we have a subgroup called -axis- that belongs to the actual cartesian plane and a second subgroup called -tics- that draws ticks for numeration with dark blue color. The stroke-dasharray attribute is a presentation value that affects the line element and allows defining the pattern of dashes and gaps of the ticks, the first value represents dash width of 2.5, followed by a space of pixels. Also, 3 circles are drawn to indicate the 3 points of the triangle named (A,B,C) using a circle and text element. The text element defines a graphics element consisting of text given coordinates, this is also used for the axis labels in blue.

For the MathML implementation , the markup starts by adding the formula's dependent variable. Identifiers are created with the mi element and next to it, an explicit space is added with the mpadded element which will be also added everytime a space is needed around the content. The mpadded element's sizing atributes accept any possible values including increment/decrement (a length prefixed with "+" or "-") in any available unit such as "em". Following the variable the mo element allows to add the corresponding operator. Afterwards we set a mfrac element , where the first child element will be a numerical mn element as a numerator and a the second child element will be denominator. For the next term, a parent mfenced element will allow to group visually under bars its subterms, the open and close attributes will specify the grouping symbol.However the mrow element will logically group the items in a mathematical fashion. Inside this, a mo element for the determinate identifier will be placed, following a child mfenced subelement. A mtable element is used to organize the four variables with subscripts. The Subscripts element is called msub , where the second child specifies the subscripts and the first child will be the baseline. After that, the closing elements for the mtable, and mfenced are called. The same procedure is followed for the last part of the formula.
